[−][src]Macro oasis_core_runtime::runtime_api

macro_rules! runtime_api {
    (
        $($api: tt)*
    ) => { ... };
}

Macro for creating runtime API definitions.

This is a meta-macro, which generates a new macro called with_api in its place. The with_api macro can be used to invoke other macros while passing the API as an argument to that macro. The position of the argument containing the API definition is specified by using the special api token in its place.

Examples

For example, if you want to register runtime methods with the dispatcher based on the given API, and have the register_runtime_txn_methods macro available, you can invoke it with this concrete API by doing:

ⓘThis example is not tested
with_api! {
    register_runtime_txn_methods!(txn_dispatcher, api);
}

Limitations

Currently the api token can only appear as the last argument and there can be at most five arguments to the inner macro.
